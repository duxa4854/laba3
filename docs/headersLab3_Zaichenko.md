## Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTeX в документации проекта

### Что такое Markdown?
#### Markdown — это облегчённый язык разметки, предназначенный для простого и интуитивно понятного форматирования текста. Он использует специальные символы, такие как звёздочки, решётки и дефисы, для создания заголовков, списков, ссылок, таблиц и других элементов. Главное преимущество Markdown — его читаемость даже в исходном формате, что упрощает редактирование и совместную работу.
### Где он применяется?
#### Markdown широко используется в различных областях: для написания документации, README-файлов на GitHub, ведения блогов, оформления сообщений в форумах и мессенджерах, создания статических сайтов (например, через генераторы вроде Jekyll или Hugo), подготовки отчётов, заметок и даже презентаций.
### Зачем он нужен разработчикам?
#### Разработчикам Markdown позволяет быстро и удобно создавать структурированные тексты без необходимости работать с громоздкими редакторами. Он упрощает ведение технической документации, оформление задач в системах управления проектами (например, в Issues на GitHub), написание пояснений в коде и создание контента для веб-страниц. Markdown легко конвертируется в HTML, PDF и другие форматы, что экономит время и обеспечивает единообразие стиля.
